# 角色与职责

Multica Squad 只把任务路由给 leader；leader 必须通过创建并分配子 Issue 来协调成员，不能假设 roster 会自动并行运行。

## 需求组

- **产品负责人**：leader；处理 stage barrier，检查阶段产物并推进下一 stage。
- **业务分析师**：范围、业务规则、异常路径、非功能需求和验收标准；作为其交付 Issue assignee 时执行需求 PR 的批准、合并和关单门禁。
- **交互原型设计师**：可运行原型、页面状态、交互说明和截图；作为其交付 Issue assignee 时执行原型 PR 的批准、合并和关单门禁。

## 开发组

- **技术负责人**：leader；拆分、集成、检查和 PR 交付，并在获得最终合并授权后执行开发 PR 合并与关单。
- **功能开发工程师**：前端、后端或全栈实现。
- **测试与持续集成工程师**：开发测试、构建检查和验收映射。
- **数据库与基础设施工程师**：schema、迁移、索引、IaC、配置、容量、备份和回滚。

## 代码评审组

- **代码评审负责人**：leader；按 PR head SHA 去重、协调评审并通过 `gh` 写回 PR；不把评审通过自行解释为合并授权。
- **代码质量评审员**：正确性、边界、接口、并发、错误处理和可维护性。
- **安全与可靠性评审员**：安全、性能、稳定性、迁移、可观测性和运维风险。

## QA/发布组

- **质量与发布负责人**：leader；QA、缺陷回流、UAT、发布和验证门禁；只在完整人工授权后推进发布。
- **功能测试工程师**：功能、边界、异常、兼容性和探索性测试。
- **自动化测试工程师**：集成、E2E、回归、性能基线和测试稳定性。
- **发布工程师**：候选构建、发布清单、迁移顺序、部署触发和回滚准备。
- **可观测性工程师**：健康、日志、指标、告警和业务指标观察。

## 人类决策权

以下决策只能由获得授权的人类作出：

1. 确认需求版本。
2. 选择实施哪些评审建议。
3. UAT 结论。
4. 生产发布批准。
5. 高风险操作、不可逆迁移、生产数据处理和回滚决策。
