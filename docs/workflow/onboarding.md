# 新人接入

## 30 分钟接入清单

1. 阅读根目录 `README.md` 和 `AGENTS.md`。
2. 阅读 `docs/workflow/` 下的生命周期、角色、门禁、评审和 QA/发布规范。
3. 浏览 `docs/templates/README.md`，了解每个阶段的固定产物。
4. 确认自己在 Multica 中的角色、可分配的 Agent 和人工决策权限。
5. 领取一个低风险 Issue，按模板完成一次从分支到 Draft PR 的演练。

## 开始任务前自检

- 是否有父 Issue 和当前子 Issue？
- 当前阶段前置门禁是否已通过？
- 需求确认 commit 是什么？
- 应使用哪个模板，产物应放在哪里？
- 哪些操作需要人工批准？
- 完成后交给哪个小队或人？

## 寻求帮助

流程不清晰时先在对应 Multica Issue 评论中提问并说明已知上下文、阻塞点和建议选项。不要在不确定时默默跳过门禁。
